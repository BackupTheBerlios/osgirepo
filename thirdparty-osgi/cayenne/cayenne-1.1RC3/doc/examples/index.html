Cayenne Examples

Cayenne examples should be downloaded separately from Cayenne. Examples are normally not tested as thourougly as the core Cayenne code. They are provided mostly for reference and learning purposes.

 Contributing Examples
Please contribute your own examples if you think they may be helpful to the new users or demonstrate some unobvious use of Cayenne.

The following examples are currently available:

Example Description
Tapestry Blog This is a fully functional (though simple) Cayenne/Tapestry blogging application.
HRDataView Demonstrates how to use Cayenne DataViews to simplify creation of Swing applications by taking advantage of rich metadata available via Cayenne.
DataPort Demonstrates how Cayenne can be used to port relational data from one database to another. It shows the operation of the few internal parts of Cayenne, normally not used in the applications. DataPort also contains a working Ant task, so it has practical value for certain database maintenance tasks.
EJB Facade Demonstrates deploying Cayenne on JBoss server and accessing it via a SessionBean facade.
Servlet Filter This is a contribution by Gary Jarrel to provide WebApplicationContextProvider functionality (i.e. automatic binding of session DataContext to a thread) via a different servlet container hook - a filter - that is compatible with Servlet specification 2.3. This will be included in the core Cayenne in the future release.
SQL Aggregate Functions Demonstrates how to use "derived" DbEntities for SQL aggregate functions, such as MIN, MAX, COUNT. With introduction of SQLTemplate this example should be considered deprecated, as derived DbEntities are very limited by nature.

Installing and Building Examples

Most examples are distributed in a source form only, users must compile them using Ant buildfiles (build.xml) provided with each example. This is generic instructions for building a given example. Some examples may require slightly different steps due to their nature.

Extract contents of cayenne-examples-[version].tar.gz into Cayenne installation directory (the location is important for the ant build files to find Cayenne): 

# cd $CAYENNE_HOME
# gz -dc cayenne-examples-[version].tar.gz | tar xf -

Go to examples directory and modify "default.properties" file that is shared by all examples. This file sets the location of Cayenne installation and JDBC driver.

If a given example contains cayenne.xml file, open it using CayenneModeler, set appropriate connection information for the DataNodes, and generate database schema.

Build example with ant:

# cd $CAYENNE_HOME/cayenne-examples-[version]/[example-name]
# ant
