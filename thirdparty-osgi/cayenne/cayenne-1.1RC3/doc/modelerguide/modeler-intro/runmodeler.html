1. Introduction


1.1. Running CayenneModeler

CayenneModeler is a regular Java application and can be run on a JSDK1.4 compatible JVM. Main class is org.objectstyle.cayenne.modeler.Main. For convenience Cayenne distribution provides run scripts to start modeler on Windows and UNIX platforms. To take advantage of these scripts, user must set CAYENNE_HOME (optional) and JAVA_HOME environment variables as described in the installation section.

To run CayenneModeler, simply doubleclick an appropriate script file. To run them from the command line, use the following commands.

Windows:

%CAYENNE_HOME%\bin\modeler.bat

UNIX:

$CAYENNE_HOME/bin/modeler.sh

 System CLASSPATH Note:
CayenneModeler startup scripts currently append System CLASSPATH variable value to the Modeler startup CLASSPATH. This behavior is obsolete and can cause a number conflicts (e.g. if a user machine has different versions of libraries used by CayenneModeler). A reference to the system CLASSPATH will be removed in the future. Currently if you run into one of such conflicts, you should remove it yourself by manually editing a script file. If you need to make Modeler aware of your external JAR files (such as JDBC drivers), there is a better way - go to "Tools > Preferences" and configure them under "ClassPath".
