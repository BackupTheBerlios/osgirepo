2. Installation


2.2 Installation

Installation of Cayenne is simple. It consists of downloading the distribution, unpacking it and making JAR files available to the applications.

Download Cayenne from http://objectstyle.org/cayenne/download.html. Choose the type of download that fits your needs. If you need a stable version, download a release distribution. To get the latest features, download a nightly build distribution. Downloaded file is called something like cayenne-src-[release-suffix].tar.gz.

 Note:
Cayenne nightly builds all pass extensive automated unit testing. But still nightly build users must understand that this is development code and it may not always work as advertised.

Unpack cayenne-[release-suffix].tar.gz into the target installation directory. On Windows you can use WinZip to unpack the distribution. WinZip supports UNIX .tar.gz archives, therefore Cayenne is not provided as a zip file. On UNIX, run the following command:

# gz -dc [filename].tar.gz | tar xf -

or if you happen to have GNU tar

# tar xzf [filename].tar.gz

To take advantage of the scripts located in the bin directory of Cayenne, set the following environment variables:

  • JAVA_HOME - should point to your JSDK 1.4 installation. It may already exist in the environment.
  • CAYENNE_HOME (optional) - should point to the directory created when your unpacked Cayenne distribution in the previous steps. (Directory named like cayenne-[release-suffix]). If CAYENNE_HOME is undefined, scripts can still guess it as long as they are run from their own directory (cayenne-[release-suffix]/bin/)
Note that these variables are only necessary to use the scripts in the bin directory. They are not needed on a deployment machine.
